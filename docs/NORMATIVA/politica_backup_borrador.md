# Politica de backup (borrador)

Ultima actualizacion: 2026-03-30

## Objetivo

Establecer una base minima para proteger informacion institucional y reducir riesgos ante borrado accidental, fallas de equipo o perdida de acceso.

## Alcance

Aplica a documentacion administrativa, pedagogica y tecnica que la institucion considere relevante conservar.

## Principios base

- La informacion importante no debe existir en una sola computadora.
- Debe haber al menos una copia adicional en otra ubicacion.
- Los respaldos deben revisarse periodicamente.
- Debe poder identificarse que version es la mas reciente.

## Esquema minimo sugerido

1. Archivo de trabajo en carpeta institucional activa.
2. Copia de respaldo en otra ubicacion segura.
3. Revision periodica de que el backup realmente abre y esta completo.

## Frecuencia sugerida

- Documentos criticos: semanal o cuando haya cambios importantes.
- Material operativo general: quincenal o mensual segun volumen.
- Scripts y documentacion tecnica: cada vez que haya cambios relevantes.

## Buenas practicas

- Usar nombres de archivo claros y consistentes.
- Evitar multiples versiones sin fecha o sin referencia.
- Guardar exportaciones PDF de documentos cerrados o definitivos.
- Mantener una persona o rol responsable de revisar respaldos.

## Recuperacion

Cuando sea posible, documentar tambien:

- donde esta el backup,
- quien puede acceder,
- como se restaura,
- como se valida que la recuperacion fue correcta.

## Estado del documento

Borrador inicial para definir una politica institucional real.
